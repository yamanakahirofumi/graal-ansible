# モジュールの開発方針 (Module Development Policy)

本プロジェクトにおける Ansible モジュールの開発および実行に関する基本方針を定義します。

## 1. モジュールの実行方式

本プロジェクトでは、**Ansible 本体の Python モジュールをそのまま（GraalPy 上で）実行する**ことを原則とします。

- **GraalPy Module**: 本家 Ansible に含まれる、またはサードパーティのコレクションとして提供される Python モジュールを、GraalPy 環境を用いて実行します。
- これにより、本家 Ansible との高度な互換性を維持し、既存の資産を最大限に活用します。

## 2. モジュールの再実装について

本プロジェクトにおいて、**Ansible モジュールの Java 等による再実装は原則として禁止（NG）**とします。

- **再実装禁止の理由**:
    - 本家 Ansible の頻繁なアップデートに追従するためのメンテナンスコストを最小限に抑えるため。
    - 独自実装による微妙な挙動の差異（バグ）の混入を防ぎ、信頼性を確保するため。
    - GraalPy による Python モジュールの直接実行を優先することで、開発リソースを実行エンジン（Core）の最適化に集中させるため。

## 3. 例外規定

パフォーマンス上の理由や、Native Image 化に伴う技術的な制約により、どうしても Java でのネイティブ実装が必要な場合は、アーキテクチャ設計会議等で個別に検討し、承認を得る必要があります。ただし、その場合でも可能な限り Python 版との互換性を厳密に検証する必要があります。
