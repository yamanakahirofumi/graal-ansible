## 2. 実装時にブレる恐れがある仕様 (実装上の留意点)

### 2.1 グループ変数の階層構造
- **懸念点**: `all` グループや親子関係を持つグループ（`children`）における、変数の継承とオーバーライドの計算順序。
- **方針**: 親グループよりも子グループの変数を優先する標準的な挙動を実装しますが、深い階層構造における計算コストを抑えるためのキャッシュ機構の有無を検討する必要があります。

### 2.2 ホストパターンのマッチング
- **懸念点**: `web*:db` や `all[0-5]` などの複雑なホスト指定パターンの解析。
- **方針**: まずは単純なホスト名、グループ名、`all` のみをサポートし、正規表現やレンジ指定は優先度を下げて実装します。

### 2.3 インベントリの自動検出
- **懸念点**: デフォルトの `/etc/ansible/hosts` や、カレントディレクトリの `hosts` ファイルをどの順序で探索するか。
- **方針**: `graal-ansible` では環境依存を避けるため、`-i` オプションによる明示的な指定を推奨し、デフォルトの探索パスは最小限に留めます。
