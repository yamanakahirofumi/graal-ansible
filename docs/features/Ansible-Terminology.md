# Ansible 用語集 (Ansible Terminology)

本ドキュメントでは、`graal-ansible` において使用される Ansible の主要な用語とその定義、および本プロジェクトでの扱いについてまとめます。

## 1. 基本コンセプト

### 1.1 コレクション (Collections)
Ansible のコンテンツ（モジュール、プラグイン、ロール、プレイブックなど）をパッケージ化した配布形式です。
- **定義**: `namespace.collection_name` という形式で識別されます。
- **本プロジェクトでの扱い**: `graal-ansible` では、実際の Ansible コレクションを GraalPy 環境にロードして使用することを基本方針としています。

### 1.2 モジュール (Modules)
リモートホストまたはローカルで実行される、特定のタスクを実行するための実行ユニットです（例: `apt`, `copy`, `service`）。
- **定義**: 冪等性（何度実行しても同じ結果になること）を持つことが推奨される、Ansible の最小実行単位です。
- **本プロジェクトでの扱い**: 本家 Ansible の Python モジュールを GraalPy 上でそのまま実行します。Java による再実装は原則として行わず、最高水準の互換性を確保します。詳細は [モジュール互換性](Module-Compatibility.md) を参照してください。

### 1.3 ロール (Roles)
関連するタスク、変数、ファイル、テンプレートなどを特定のディレクトリ構造にまとめた再利用可能なコンポーネントです。
- **定義**: 複雑な設定を抽象化し、複数のプレイブックで共有できるようにしたものです。
- **本プロジェクトでの扱い**: 標準的なロール構成（`tasks/`, `vars/`, `defaults/`, `handlers/` など）をサポートし、プレイブックからの呼び出しを可能にします。

## 2. 実行構成要素

### 2.1 プレイブック (Playbook)
YAML 形式で記述された、自動化手順の定義ファイルです。
- **定義**: 1つ以上の「プレイ (Play)」で構成され、どのホストに対してどのタスクを実行するかを記述します。
- **本プロジェクトでの扱い**: `SnakeYAML` を用いて解析し、Java 側の実行エンジンで処理します。詳細は [Playbook 実行仕様](Playbook-Execution.md) を参照してください。

### 2.2 プレイ (Play)
プレイブック内の構成単位で、特定のホストグループ（インベントリで定義）に対してタスクをマッピングする役割を持ちます。

### 2.3 タスク (Task)
プレイ内での最小実行ステップです。特定のモジュールを呼び出し、引数を渡します。

### 2.4 ハンドラー (Handlers)
タスクの実行結果によって状態が変化した（`changed`）場合にのみ呼び出される特殊なタスクです。サービスの再起動などに利用されます。

## 3. 環境・管理要素

### 3.1 インベントリ (Inventory)
管理対象となるホストやグループの情報を定義したリストです。
- **定義**: 静的なファイル（INI, YAML 形式）や、動的なスクリプトによって生成されます。
- **本プロジェクトでの扱い**: INI 形式および YAML 形式の静的インベントリを優先的にサポートします。詳細は [インベントリ管理](Inventory-System.md) を参照してください。

### 3.2 変数 (Variables)
設定値を動的に管理するための仕組みです。インベントリ、プレイブック、ロールなど様々な場所で定義可能です。
- **本プロジェクトでの扱い**: Ansible 本家の複雑な優先順位を段階的に実装します。詳細は [Playbook 実行仕様](Playbook-Execution.md) の「変数のスコープと優先順位」を参照してください。

### 3.3 ファクト (Facts)
Ansible が実行時にリモートホストから自動的に収集するシステム情報（OS種別、IPアドレス、メモリ量など）です。
- **本プロジェクトでの扱い**: `setup` モジュール等を通じて収集されるファクトを、変数として利用できるようにします。

## 4. ノード構成

### 4.1 管理ノード (Control Node)
Ansible を実行するマシンです。
- **定義**: プレイブックの実行エンジン、インベントリ、設定ファイルなどが配置され、コマンドを実行する環境です。
- **本プロジェクトでの扱い**: `graal-ansible` (Java) および GraalPy が動作する環境が管理ノードとなります。

### 4.2 管理対象ノード (Managed Node / Target Node)
Ansible によって管理されるターゲットマシン（ホスト）です。
- **定義**: インベントリで定義され、管理ノードから接続してタスクが実行される対象です。
- **本プロジェクトでの扱い**: SSH 等で接続可能なリモートホスト、または `local` 接続時の自分自身が対象となります。

### 4.3 接続プラグイン (Connection Plugins)
管理ノードから管理対象ノードへの接続手段を提供するプラグインです（例: `ssh`, `local`, `docker`）。
- **本プロジェクトでの扱い**: `ssh` および `local` 接続を優先的にサポートします。詳細は [接続システム](Connection-System.md) を参照してください。
