# CLI仕様

`graal-ansible` は、`ansible-playbook` と互換性のあるコマンドラインインターフェースを提供することを目指します。

## 1. 基本コマンド形式

```bash
graal-ansible [options] playbook.yml
```

## 2. サポート予定の主要オプション

| オプション | 短縮形 | 説明 | 優先度 |
| :--- | :--- | :--- | :--- |
| `--inventory` | `-i` | インベントリファイルのパスを指定 | 高 |
| `--extra-vars` | `-e` | 追加の変数を設定 (key=value または JSON/YAML) | 高 |
| `--limit` | `-l` | 実行対象のホストを制限 | 中 |
| `--tags` | `-t` | 特定のタグが付いたタスクのみ実行 | 中 |
| `--skip-tags` | - | 特定のタグが付いたタスクをスキップ | 低 |
| `--check` | `-C` | 変更を加えずに実行（ドライラン） | 中 |
| `--diff` | `-D` | ファイルの変更内容を表示 | 低 |
| `--verbose` | `-v` | 詳細ログを表示 (`-vvv` 等の複数指定をサポート予定) | 高 |
| `--version` | - | バージョン情報を表示 | 高 |
| `--help` | `-h` | ヘルプメッセージを表示 | 高 |

## 3. 実装方針

- **解析ライブラリ**: [picocli](https://picocli.info/) の使用を検討。
- **Native Image 対応**: GraalVM Native Image での実行時にリフレクション設定が必要になるため、picocli のアノテーションプロセッサを活用する。
- **互換性**: 戻り値（終了コード）についても Ansible 本家と同一の仕様とする。
    - `0`: 正常終了
    - `1`: 一般的なエラー
    - `2`: ホストの一部が失敗
    - `4`: 構文エラー
