# タスク制御の実装詳細 (Task Control Implementation)

本ドキュメントでは、`graal-ansible` におけるタスクの実行制御（条件分岐、ループ、変数登録、ハンドラー）の実装方法について詳述します。

## 1. 条件分岐 (`when`)

タスクの実行を特定の条件に基づいて制御します。

- **実装方針**:
    - `when` 句に記述された Jinja2 式を `VariableResolver` (Jinjava) を用いて評価します。
    - 評価結果が `true` または真とみなされる値（非空の文字列、非ゼロの数値等）の場合のみ、タスクを実行します。
    - 評価結果が `false` の場合、そのホストでのタスク実行はスキップされ、結果には `skipped=true` が含まれます。
    - `when` 句がリスト形式で記述されている場合は、すべての条件が `true` であること（論理積）を確認します。
- **留意点**:
    - Jinja2 の式として評価するため、変数は `{{ }}` で囲まずに記述されることが一般的であることに対応します。
- **データ構造**:
    - `Task` レコードに `String when` フィールドを追加します。

## 2. ループ (`loop`)

同一のタスクを複数のアイテムに対して繰り返し実行します。

- **実装方針**:
    - `loop` フィールドに指定されたリスト（またはリストを返すテンプレート）を解析します。
    - リストの各アイテムについて、特殊変数 `item` にその値を割り当て、タスクを繰り返し実行します。
    - 各イテレーションの結果は、最終的なタスク結果の `results` リストに格納されます。
- **レガシー対応**:
    - `with_items` 等のレガシーなループ構文は、内部的に `loop` 処理へ変換することを検討します。
- **データ構造**:
    - `Task` レコードに `Object loop` フィールドを追加します。

## 3. 変数登録 (`register`)

タスクの実行結果を後のタスクで利用するために変数として保存します。

- **実装方針**:
    - タスク実行後、`register` キーで指定された名前で、実行結果（`TaskResult` のデータ部分）を `VariableManager` に登録します。
    - 登録された変数は、同一 Play 内の以降のタスクで参照可能です。
- **データ構造**:
    - `Task` レコードに `String register` フィールドを追加します。
    - 登録される変数は Map 形式とし、Ansible 互換のフィールド名（`rc`, `stdout`, `stderr`, `changed`, `failed` 等）を保持します。

## 4. ハンドラーと通知 (`handlers`, `notify`)

タスクによって状態が変更された（`changed=true`）場合にのみ実行される特別なタスクを制御します。

- **実装方針**:
    - **通知 (Notify)**: タスクが実行され、結果が `changed=true` かつ `notify` 句がある場合、指定されたハンドラー名を「実行待ちキュー」に登録します。
    - **実行タイミング**: プレイ（Play）内のすべての通常タスクが完了した直後、または `flush_handlers` タスクが実行されたタイミングで、キューに溜まったハンドラーを順次実行します。
    - **冪等性**: ハンドラーは、一つのプレイ内で何度通知されても、最後に一度だけ実行されます。
- **データ構造**:
    - `Task` レコードに `List<String> notify` フィールドを追加します。
    - `Play` レコードに `List<Task> handlers` フィールドを追加します。

## 5. 実行エンジン (`PlaybookExecutor`) での処理順序

各タスクの実行時、以下の順序で制御ロジックを適用します。

1. **ループの展開**: `loop` がある場合、リストを取得しイテレーションを開始します。
2. **条件評価**: `when` 句を評価し、実行の可否を判断します。
3. **タスク実行**: `TaskExecutor` を通じてモジュールを実行します。
4. **結果登録**: `register` がある場合、結果を保存します。
5. **通知**: `changed=true` かつ `notify` がある場合、ハンドラーをマークします。
