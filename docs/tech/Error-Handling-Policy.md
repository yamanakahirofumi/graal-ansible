# エラーハンドリング方針

本プロジェクトにおけるエラーハンドリングの基本方針と、具体的な各ケースでの対応を定義します。

## 1. 基本方針

- **実行継続性の制御**：Playbook実行中にエラーが発生した場合、対象ホストの実行を停止するか（any_errors_fatal）、次のタスクに進むかをAnsibleの仕様に従って制御します。
- **詳細なエラー原因の提示**：YAMLの構文エラーや、リモートホストでの実行失敗時に、ユーザーが原因を特定できる詳細な情報を出力します。
- **例外設計**：
    - YAML解析エラー、変数展開エラー、接続エラーなど、エラーの種類に応じたカスタム例外を定義します。
    - Native Imageでの実行を考慮し、スタックトレースの保持やシリアライズの制約に留意します。

## 2. 具体的なシナリオと対応

### 2.1 Playbook 解析時
- **YAML構文エラー**：行番号とエラー箇所を特定し、ユーザーに通知します。
- **スキーマ違反**：必須パラメータの欠落や不正なデータ型を検知し、実行前にエラーを出力します。

### 2.2 タスク実行時
- **モジュール実行失敗**：戻り値の `failed` フラグを真とし、エラーメッセージを収集します。
- **接続タイムアウト**：SSH接続等の失敗を適切に検知し、リトライまたはホストの除外を行います。
