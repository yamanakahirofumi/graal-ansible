# 品質方針（Quality Policy）

本ドキュメントは、ansible-coreのJava再実装プロジェクトにおける品質の考え方と、OS非依存性の検証基準を定義します。

## 1. 基本方針
- **OS非依存性の実証**：Windows, Linux, macOSの各OSで同一のPlaybookが期待通りに動作することを最優先事項とします。
- **互換性の重視**：本家ansible-coreとの動作互換性を可能な限り維持し、既存のAnsible資産を活用可能にします。
- **Native Imageの安定性**：GraalVM Native Image化されたバイナリが、JIT実行時と同等の信頼性で動作することを保証します。

## 2. 検証軸
本プロジェクトでは以下の3つの軸で品質を検証します。

### 2.1 機能的正確性
- PlaybookのYAML解析が正確であること。
- **GraalPyによるPythonモジュール実行**：既存のPython製モジュールがGraalVM上で正しく動作すること。
- 各モジュール（file, command等）が指定されたパラメータに従って正しく状態を変更すること。
- 変数展開や条件分岐がAnsibleの仕様通りに動作すること。

### 2.2 OS非依存性（クロスプラットフォーム検証）
- **ファイルパス処理**：スラッシュ/バックスラッシュの扱いやドライブレターが適切に抽象化されていること。
- **プロセス実行**：シェル呼び出しやバイナリ実行が各OSの作法に従いつつ、同一のインターフェースで提供されること。
- **パーミッション**：POSIX権限とWindows ACLの差異が、モジュール仕様の範囲内で適切にマッピングされていること。

### 2.3 実行形態の妥当性
- **JIT実行 (Standard JVM)**：通常のJava実行環境での安定動作。
- **AOT実行 (Native Image)**：Native Image特有の制約（リフレクション、リソースアクセス等）をクリアし、高速に動作すること。

## 3. テスト戦略
- **ユニットテスト**：個別のロジック（YAML解析、変数計算等）をJava上で高速に検証。
- **結合テスト (localhost)**：各OS環境のローカルマシンに対して実際にモジュールを実行し、状態変化を検証。
- **互換性テスト**：本家Ansibleと同一のPlaybookを実行し、結果（JSON出力等）を比較検証。

## 4. 品質ゲート
- 全てのユニットテストがグリーンであること。
- サポート対象とする全OS（CI上のUbuntu, Windows, macOS）で結合テストが成功すること。
- Native Imageビルドが成功し、生成されたバイナリで基本Playbookが完遂すること。
