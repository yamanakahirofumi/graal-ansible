# ロギング方針

デバッグおよび保守のため、適切なロギングを行います。

## 1. 基本方針
本プロジェクトでは、Java 標準のロギング API を使用して、アプリケーションの動作状況や例外発生時の情報を記録します。

## 2. 使用ライブラリ
- **ライブラリ**：`java.util.logging` (標準)

## 3. ログレベルの使い分け
ログレベルは以下のように使い分けます。

- **INFO**：
    - Playbookの実行開始/終了、Play/Taskの開始、インベントリの読み込み完了。
    - ユーザーに標準出力される実行状況の記録。
- **WARNING**：
    - 非推奨のモジュール使用、不適切な設定値（だが実行可能）、接続リトライの発生。
- **SEVERE**：
    - Playbook解析不能、致命的な接続失敗、OS互換性の欠如による実行停止。

## 4. 出力内容
- 発生日時
- ログレベル
- 発生箇所（クラス名、メソッド名）
- メッセージ（および例外発生時はスタックトレース）
